## 社区

### 💭 GitHub Discussions

我们有一个 [discussions](https://github.com/langchain-ai/langchain/discussions) 页面，用户可以在这里提问使用问题、讨论设计决策以及提议新功能。

如果你能够回答问题，请这样做！这将使维护者能够将更多时间专注于开发和 bug 修复。

### 🚩 GitHub Issues

我们的 [issues](https://github.com/langchain-ai/langchain/issues) 页面会及时更新 bug、改进和功能请求。

有一个 [标签分类](https://github.com/langchain-ai/langchain/labels?sort=count-desc)
有助于对感兴趣的 issues 进行排序和发现。请使用这些标签来帮助
组织 issues。查看 [需要帮助 (Help Wanted)](https://github.com/langchain-ai/langchain/labels/help%20wanted)
和 [第一个简单问题 (Good First Issue)](https://github.com/langchain-ai/langchain/labels/good%20first%20issue)
标签以获取建议。

如果你开始处理一个 issue，请将其自分配给你自己。

如果你正在添加一个 issue，请尝试将其集中在一个单独的、模块化的 bug/改进/功能上。
如果两个 issues 相关或相互依赖，请链接它们而不是合并。

我们将尽量保持这些 issues 的最新状态，尽管
由于该领域的快速发展，其中一些可能已经过时。
如果你注意到这种情况，请告知我们。

### 📢 社区 Slack

我们有一个 [community slack](https://www.langchain.com/join-community)，你可以在其中提问、获得帮助以及与其他贡献者和用户讨论项目。

### 🙋 获取帮助

我们的目标是拥有尽可能简单的开发者设置。如果你在设置过程中遇到任何困难，请
在 [community slack](https://www.langchain.com/join-community) 中提问或在 GitHub 上开启一个 [discussion](https://github.com/langchain-ai/langchain/discussions)。

同样，我们对代码库强制执行某些代码风格、格式设置和文档标准。
如果你觉得这些很难（甚至很烦人）与之配合，请随时在 [community slack](https://www.langchain.com/join-community) 中提问！
