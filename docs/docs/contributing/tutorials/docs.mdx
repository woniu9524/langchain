# 提交你的第一个文档 PR

本教程将指导你完成一个简单的文档编辑，例如更正拼写错误。

### **先决条件**
- GitHub 账户。
- 熟悉 [GitHub 拉取请求](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/about-pull-requests)（基本了解）。

---

## 在 GitHub 上编辑文档页面

有时你可能只想做一个小的改动，比如修复一个拼写错误，最简单的方法就是直接使用 GitHub 的编辑器。

### **步骤**

1. **导航到 LangChain 文档中的相应页面：**
   - 在文档页面上，找到页面底部的绿色“Edit this page”链接。
   - 点击该链接即可跳转到 GitHub 编辑器。
   - 如果你要编辑的文件是 Jupyter Notebook（.ipynb）而不是 Markdown（.md, .mdx）文件，我们建议遵循第 3 部分的步骤。

2. **Fork 该仓库：**
   - 如果你还没有这样做，GitHub 会提示你将该仓库 fork 到你的账户下。
   - 确保将仓库 fork 到你的**个人账户而不是组织**（[原因？](../reference/faq.mdx#how-do-i-allow-maintainers-to-edit-my-pr)）。
   - 点击“Fork this repository”按钮，在你的账户下创建一个该仓库的副本。
   - Fork 之后，你将自动重定向到正确的编辑器。

3. **进行更改：**
   - 直接在 GitHub 编辑器中更正拼写错误。

4. **提交更改：**
   - 点击页面右上角的“Commit changes...”按钮。
   - 为你的提交起一个标题，例如“Fix typo in X section”（修复 X 部分的拼写错误）。
   - 可选地，撰写一个扩展的提交描述。
   - 点击“Propose changes”（提议更改）。

5. **提交拉取请求 (PR)：**
   - GitHub 会将你重定向到一个可以创建拉取请求的页面。
   - 首先，请检查你提议的更改，确保它们是正确的。
   - 点击 **Create pull request**（创建拉取请求）。
   - 为你的 PR 起一个标题，例如 `docs: Fix typo in X section`（docs: 修复 X 部分的拼写错误）。
   - 遵循 PR 描述模板中的清单。

## 获取评审

提交拉取请求后，维护者会对其进行评审。你可能会收到反馈或修改意见。请留意 PR 以处理任何评论。

文档 PR 通常会在几天内得到评审，但具体时间可能因更改的复杂性和维护者的可用性而异。

有关评审的更多信息，请参阅 [评审流程](../reference/review_process.mdx)。
