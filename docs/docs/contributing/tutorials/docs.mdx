# 提交你的第一个 Docs PR

本教程将指导你完成一个简单的文档编辑，例如纠正拼写错误。

### **先决条件**
- GitHub 账户。
- 熟悉 [GitHub Pull Requests](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/about-pull-requests)（基本了解）。

---

## 在 GitHub 上编辑文档页面

有时你只想做一个小的改动，比如修正一个拼写错误，而最简单的方法就是直接使用 GitHub 的编辑器。

### **步骤**

1. **导航到 LangChain 文档中的相应页面：**
   - 在文档页面上，找到页面底部的绿色“Edit this page”链接。
   - 点击该按钮即可被重定向到 GitHub 编辑器。
   - 如果你要编辑的文件是 Jupyter Notebook（.ipynb）而不是 Markdown（.md, .mdx）文件，我们建议遵循第 3 部分的步骤。

2. **Fork 代码仓库：**
   - 如果你还没有这样做，GitHub 会提示你将代码仓库 fork 到你的账户。
   - 确保将代码仓库 fork 到你的 **个人账户而不是组织**（[原因？](../reference/faq.mdx#how-do-i-allow-maintainers-to-edit-my-pr)）。
   - 点击“Fork this repository”按钮，在你的账户下创建一个代码仓库的副本。
   - Fork 完成后，你会自动被重定向到相应的编辑器。

3. **进行更改：**
   - 直接在 GitHub 编辑器中修正拼写错误。

4. **提交你的更改：**
   - 点击页面右上角的“Commit changes...”按钮。
   - 给你的提交一个标题，例如“Fix typo in X section.”。
   - 可以选择性地写一个更详细的提交描述。
   - 点击“Propose changes”。

5. **提交一个 Pull Request (PR)：**
   - GitHub 会将你重定向到一个可以创建 Pull Request 的页面。
   - 首先，检查你建议的更改，确保它们是正确的。
   - 点击 **Create pull request**。
   - 给你的 PR 一个标题，例如 `docs: Fix typo in X section`。
   - 遵循 PR 描述模板中的清单。

## 获取评审

提交 Pull Request 后，它将由维护者进行评审。你可能会收到反馈或更改请求。请留意 PR 以处理任何评论。

Docs PR 通常在几天内完成评审，但具体时间可能因更改的复杂性和维护者的可用性而异。

有关评审的更多信息，请参阅 [评审流程](../reference/review_process.mdx)。
