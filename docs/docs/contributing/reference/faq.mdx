# 常见问题解答

## 拉取请求 (PRs)

### 如何允许维护者编辑我的 PR？

当您提交拉取请求时，在合并之前可能还需要进行其他更改。通常，由维护者在合并前进行这些更改比要求您在代码审查中进行这些更改更有效率。

默认情况下，大多数拉取请求在右侧边栏都会有一个
`✅ 允许维护者编辑此拉取请求。`
徽章。

如果您没有看到此徽章，您可能已为从中拉取请求的分叉关闭了此设置。有关更多信息，请参阅 [此 Github 文档页面](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/allowing-changes-to-a-pull-request-branch-created-from-a-fork)。

值得注意的是，Github 不允许在**组织**中为分叉启用此设置（[issue](https://github.com/orgs/community/discussions/5634)）。
如果您在组织中工作，我们建议从个人分叉提交您的 PR 以启用此设置。

### 为什么我的 PR 没有得到审查？

请参考我们的 [审查流程](review_process.mdx)。

### 为什么我的 PR 被关闭了？

请参考我们的 [审查流程](review_process.mdx)。

### 我认为我的 PR 被关闭的方式不符合审查流程。我该怎么办？

在 PR 评论中标记 `@ccurme`，并引用您认为未遵循的审查流程部分。我们会查看！
