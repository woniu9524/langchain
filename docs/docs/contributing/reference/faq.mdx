# 常见问题解答

## 拉取请求 (PRs)

### 如何允许维护者编辑我的拉取请求？

提交拉取请求后，可能需要进行其他更改才能合并。通常，由维护者在合并前自行进行这些更改比要求你在代码审查中进行这些更改更有效率。

默认情况下，大多数拉取请求在右侧边栏会有一个徽章
`✅ 允许维护者编辑此拉取请求。`

如果你没有看到这个徽章，你可能没有为你请求合并的分支（fork）启用此设置。请参阅 [此 Github 文档页面](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/allowing-changes-to-a-pull-request-branch-created-from-a-fork) 了解更多信息。

值得注意的是，Github 不允许在**组织**内的分支（fork）启用此设置（[issue](https://github.com/orgs/community/discussions/5634)）。
如果你在组织内工作，我们建议从个人分支（fork）提交你的拉取请求以启用此设置。

### 为什么我的拉取请求没有被审查？

请参考我们的 [审查流程](review_process.mdx)。

### 为什么我的拉取请求被关闭了？

请参考我们的 [审查流程](review_process.mdx)。

### 我觉得我的拉取请求关闭的方式不符合审查流程。我该怎么办？

在拉取请求评论中标记 `@ccurme`，并引用你认为未遵循的审查流程部分。我们会进行查看！
