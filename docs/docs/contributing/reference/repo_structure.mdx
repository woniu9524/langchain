# 仓库结构

如果你打算为 LangChain 的代码或文档做贡献，了解仓库的整体结构会很有帮助。

LangChain 被组织成一个包含多个包的 [monorepo](https://en.wikipedia.org/wiki/Monorepo)。
你可以查看我们的 [安装指南](/docs/how_to/installation/) 来了解它们是如何协同工作的。

下面是仓库结构的树状可视化：

```text
.
├── cookbook # 教程和示例
├── docs # 包含文档内容：https://python.langchain.com/
├── libs
│   ├── langchain
│   │   ├── langchain
│   │   ├── tests/unit_tests # 单测 (每个包中都存在，为简洁起见未显示)
│   │   ├── tests/integration_tests # 集成测试 (每个包中都存在，为简洁起见未显示)
│   ├── community # 第三方集成
│   │   ├── langchain-community
│   ├── core # 关键抽象的基础接口
│   │   ├── langchain-core
│   ├── experimental # 实验性组件和链
│   │   ├── langchain-experimental
|   ├── cli # 命令行接口
│   │   ├── langchain-cli
│   ├── text-splitters
│   │   ├── langchain-text-splitters
│   ├── standard-tests
│   │   ├── langchain-standard-tests
│   ├── partners
│       ├── langchain-partner-1
│       ├── langchain-partner-2
│       ├── ...
│
├── templates # 一系列易于部署的参考架构，适用于各种任务。
```

根目录还包含以下文件：

* `pyproject.toml`: 构建文档和 linting 文档、cookbook 的依赖项。
* `Makefile`: 包含构建、linting、文档和 cookbook 的快捷方式的文件。

根目录级别还有其他文件，但它们的存在应该是显而易见的。欢迎随意浏览！

## 文档

`/docs` 目录包含显示在 https://python.langchain.com/ 的文档内容以及相关的 API 参考 https://python.langchain.com/api_reference/langchain/index.html。

请参阅 [文档](../how_to/documentation/index.mdx) 指南，了解如何为文档做出贡献。

## 代码

`/libs` 目录包含 LangChain 包的代码。

要了解更多关于如何贡献代码的信息，请参阅以下指南：

- [代码](../how_to/code/index.mdx): 了解如何在 LangChain 代码库中进行开发。
- [集成](../how_to/integrations/index.mdx): 了解如何为 `langchain-community` 贡献第三方集成或启动新的合作伙伴包。
- [测试](../how_to/testing.mdx): 用于了解如何为包编写测试的指南。
