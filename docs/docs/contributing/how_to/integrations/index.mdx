# 贡献集成

集成是 LangChain 的核心组成部分。
LangChain 为构建 LLM 应用至关重要的几个不同组件（语言模型、向量存储等）提供了标准接口。

## 为什么向 LangChain 贡献集成？

- **可发现性：** LangChain 是构建 LLM 应用最常用的框架，月下载量超过 2000 万次。LangChain 集成可以被庞大的 GenAI 构建者社区发现。
- **互操作性：** LangChain 组件公开标准接口，允许开发人员轻松地相互替换。如果您实现了 LangChain 集成，任何使用不同组件的开发人员都可以轻松地替换您的组件。
- **最佳实践：** 通过其标准接口，LangChain 组件鼓励并促进最佳实践（流式传输、异步等）。

## 如何贡献集成

要贡献集成，您应该遵循以下步骤：

1. 确认您的集成在我们当前鼓励的[组件列表](#components-to-integrate)中。
2. [实现您的包](/docs/contributing/how_to/integrations/package/)并将其发布到公共 GitHub 存储库。
3. [实现标准测试](./standard_tests)以供您的集成使用，并成功运行它们。
4. 通过将包发布到 PyPi 并将文档添加到 LangChain 单体仓库的 `docs/docs/integrations` 目录中来[发布您的集成](./publish.mdx)。
5. [可选] 打开并合并一个 PR，将您的集成文档添加到官方 LangChain 文档中。
6. [可选] 与 LangChain 团队进行联合联合营销（[见下文](#co-marketing)）。

## 联合营销

LangChain 月下载量超过 2000 万次，拥有庞大的 LLM 应用构建者受众。除了列出集成之外，我们还旨在突出高质量、教育性的示例，以激励开发人员并推动生态系统发展。

虽然我们偶尔会分享集成，但我们优先考虑提供有意义的见解和最佳实践的内容。我们的主要社交渠道是 Twitter 和 LinkedIn，我们会在这些渠道上重点介绍最佳示例。

以下是我们乐于推广的内容类型的启发式方法：

- **教育内容：** 展示教育内容的博客、YouTube 视频和其他媒体。请注意，我们更喜欢不以“如何使用集成 XYZ”为框架的内容，而是以“如何做 ABC”为框架的内容，因为我们发现这对于开发人员来说更具教育性和帮助性。
- **端到端应用程序：** 端到端应用程序是寻找构建内容的开发人员的宝贵资源。我们更倾向于重点介绍更复杂/代理性质的应用程序，并使用 [LangGraph](https://github.com/langchain-ai/langgraph) 作为编排框架。我们对涉及长期记忆、人机循环交互模式或多代理架构的任何内容都特别感兴趣。
- **研究：** 我们喜欢重点介绍新颖的研究！无论是基于 LangChain 构建的研究，还是与之集成的研究。
