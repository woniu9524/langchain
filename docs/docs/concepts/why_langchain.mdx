# 为什么选择 LangChain？

`langchain` Python 包和 LangChain 公司致力于让开发者尽可能轻松地构建能够进行推理的应用程序。
虽然 LangChain 最初是一个单一的开源包，但它已经发展成为一家公司和一个完整的生态系统。
本页面将讨论 LangChain 生态系统作为一个整体。
LangChain 生态系统中的大多数组件都可以独立使用——所以如果你特别喜欢某些组件而不喜欢其他组件，那完全没问题！为你自己的用例挑选并选择你最喜欢的组件！

## 特性

LangChain 旨在解决几个主要需求：

1. **标准化的组件接口：** 随着 AI 应用程序的[模型](/docs/integrations/chat/)和[相关组件](/docs/integrations/vectorstores/)的不断增长，出现了各种不同的 API，开发者需要学习和使用它们。
这种多样性使得开发者在构建应用程序时难以在提供商之间切换或组合组件。
LangChain 为关键组件提供了标准接口，可以轻松地在提供商之间切换。

2. **编排：** 随着应用程序变得越来越复杂，将多个组件和模型组合在一起，[将这些元素有效地连接到可以[完成各种任务](https://www.sequoiacap.com/article/generative-ais-act-o1/)的控制流中](https://lilianweng.github.io/posts/2023-06-23-agent/)的需求日益增长。
[编排](https://en.wikipedia.org/wiki/Orchestration_(computing))对于构建此类应用程序至关重要。

3. **可观测性和评估：** 随着应用程序变得越来越复杂，理解应用程序内部发生的情况也变得越来越困难。
此外，开发速度可能会受到[选择悖论](https://en.wikipedia.org/wiki/Paradox_of_choice)的限制。
例如，开发者经常想知道如何设计他们的提示或哪个 LLM 最能平衡准确性、延迟和成本。
[可观测性](https://en.wikipedia.org/wiki/Observability)和评估可以帮助开发者监控他们的应用程序并自信地快速回答这些类型的问题。


## 标准化的组件接口

LangChain 为许多 AI 应用程序的核心组件提供了通用接口。
例如，所有[聊天模型](/docs/concepts/chat_models/)都实现了 [BaseChatModel](https://python.langchain.com/api_reference/core/language_models/langchain_core.language_models.chat_models.BaseChatModel.html) 接口。
这提供了一种与聊天模型交互的标准方式，支持重要的但通常是特定于提供商的功能，例如[工具调用](/docs/concepts/tool_calling/)和[结构化输出](/docs/concepts/structured_outputs/)。


### 示例：聊天模型

许多[模型提供商](/docs/concepts/chat_models/)支持[工具调用](/docs/concepts/tool_calling/)，这是许多应用程序（例如[代理](https://langchain-ai.github.io/langgraph/concepts/agentic_concepts/)）的关键功能，它允许开发者请求与特定模式匹配的模型响应。
每个提供商的 API 都不同。
LangChain 的[聊天模型](/docs/concepts/chat_models/)接口提供了一种通用的方式来绑定[工具](/docs/concepts/tools)到模型以支持[工具调用](/docs/concepts/tool_calling/)：

```python
# Tool creation
tools = [my_tool]
# Tool binding
model_with_tools = model.bind_tools(tools)
```

同样，让模型生成[结构化输出](/docs/concepts/structured_outputs/)也是一个极其常见的用例。
提供商支持不同的方法来实现这一点，包括[JSON 模式或工具调用](https://platform.openai.com/docs/guides/structured-outputs)，具有不同的 API。
LangChain 的[聊天模型](/docs/concepts/chat_models/)接口提供了一种使用 `with_structured_output()` 方法生成结构化输出的通用方法：

```python
# Define schema
schema = ...
# Bind schema to model
model_with_structure = model.with_structured_output(schema)
```

### 示例：检索器

在[RAG](/docs/concepts/rag/)和 LLM 应用程序组件的上下文中，LangChain 的[检索器](/docs/concepts/retrievers/)接口提供了一种标准方式来连接到许多不同类型的数据服务或数据库（例如，[向量存储](/docs/concepts/vectorstores)或数据库）。
检索器的底层实现取决于你连接的数据存储或数据库的类型，但所有检索器都实现了[可运行接口](/docs/concepts/runnables/)，这意味着它们可以以通用方式调用。

```python
documents = my_retriever.invoke("What is the meaning of life?")
```

## 编排

虽然单个组件的标准化很有用，但我们越来越多地看到开发者希望将组件*组合*成更复杂的应用程序。
这激发了对[编排](https://en.wikipedia.org/wiki/Orchestration_(computing))的需求。
LLM 应用程序的几个共同特征是此编排层应支持：

* **复杂的控制流：** 应用程序需要复杂的模式，例如循环（例如，一个循环直到满足条件才重复）。
* **[持久化](https://langchain-ai.github.io/langgraph/concepts/persistence/)：** 应用程序需要维护[短期和/或长期记忆](https://langchain-ai.github.io/langgraph/concepts/memory/)。
* **[人工干预](https://langchain-ai.github.io/langgraph/concepts/human_in_the_loop/)：** 应用程序需要人工交互，例如暂停、审查、编辑、批准某些步骤。

编排复杂应用程序组件的推荐方法是使用[LangGraph](https://langchain-ai.github.io/langgraph/concepts/high_level/)。
LangGraph 是一个库，它通过将应用程序流表示为一组节点和边，为开发者提供了高度的控制。
LangGraph 内置支持[持久化](https://langchain-ai.github.io/langgraph/concepts/persistence/)、[人工干预](https://langchain-ai.github.io/langgraph/concepts/human_in_the_loop/)、[记忆](https://langchain-ai.github.io/langgraph/concepts/memory/)和其他功能。
它特别适合构建[代理](https://langchain-ai.github.io/langgraph/concepts/agentic_concepts/)或[多代理](https://langchain-ai.github.io/langgraph/concepts/multi_agent/)应用程序。
重要的是，单个 LangChain 组件可以用作 LangGraph 节点，但你也可以在**不**使用 LangChain 组件的情况下使用 LangGraph。

AI 应用程序的开发速度通常受限于高质量的评估，因为存在选择悖论。
开发者经常想知道如何设计他们的提示或哪个 LLM 最能平衡准确性、延迟和成本。
高质量的跟踪和评估可以帮助你自信地快速回答这些类型的问题。

## 结论

LangChain 为许多 AI 应用程序的核心组件提供了标准接口，这带来了几个特定的优势：
- **轻松更换提供商：** 它允许你在不更改底层代码的情况下更换不同的组件提供商。
- **高级功能：** 它为[流式传输](/docs/concepts/streaming)和[工具调用](/docs/concepts/tool_calling/)等高级功能提供了通用方法。

[LangGraph](https://langchain-ai.github.io/langgraph/concepts/high_level/) 使编排复杂应用程序（例如[代理](/docs/concepts/agents/)）成为可能，并提供诸如[持久化](https://langchain-ai.github.io/langgraph/concepts/persistence/)、[人工干预](https://langchain-ai.github.io/langgraph/concepts/human_in_the_loop/)或[记忆](https://langchain-ai.github.io/langgraph/concepts/memory/)等功能。

[LangSmith](https://docs.smith.langchain.com/) 通过提供 LLM 特定的可观测性以及用于测试和评估应用程序的框架，使你能够自信地迭代你的应用程序。
