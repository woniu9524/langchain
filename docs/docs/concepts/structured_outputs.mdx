# 结构化输出

## 概述

对于许多应用程序，例如聊天机器人，模型需要直接以自然语言响应用户。
但是，在某些情况下，我们需要模型以*结构化格式*输出。
例如，我们可能希望将模型输出存储在数据库中，并确保输出符合数据库模式。
这种需求促成了结构化输出的概念，即可以指示模型以特定的输出结构进行响应。

![结构化输出](/img/structured_output.png)

## 关键概念

1. **模式定义：** 输出结构由模式表示，该模式可以通过多种方式定义。
2. **返回结构化输出：** 模型将接收此模式，并被指示返回符合该模式的输出。

## 推荐用法

此伪代码说明了使用结构化输出时推荐的工作流程。
LangChain 提供了一个方法 [`with_structured_output()`](/docs/how_to/structured_output/#the-with_structured_output-method)，该方法可自动将模式绑定到[模型](/docs/concepts/chat_models/)并解析输出。
此辅助函数适用于支持结构化输出的所有模型提供商。

```python
# 定义模式
schema = {"foo": "bar"}
# 将模式绑定到模型
model_with_structure = model.with_structured_output(schema)
# 调用模型以生成符合模式的结构化输出
structured_output = model_with_structure.invoke(user_input)
```

## 模式定义

核心概念是模型响应的输出结构需要以某种方式表示。
虽然您可以使用的对象类型取决于您正在使用的模型，但通常允许或推荐用于 Python 中结构化输出的常见对象类型。

结构化输出最简单、最常见的格式是类似 JSON 的结构，在 Python 中可以表示为字典 (dict) 或列表 (list)。
当工具需要原始、灵活且开销最小的结构化数据时，通常直接使用 JSON 对象（或 Python 中的 dict）。

```json
{
  "answer": "用户问题的答案",
  "followup_question": "用户可以提出的后续问题"
}
```

作为第二个示例，[Pydantic](https://docs.pydantic.dev/latest/) 在定义结构化输出模式方面特别有用，因为它提供了类型提示和验证。
以下是 Pydantic 模式的示例：

```python
from pydantic import BaseModel, Field
class ResponseFormatter(BaseModel):
    """始终使用此工具来格式化您对用户的响应。"""
    answer: str = Field(description="用户问题的答案")
    followup_question: str = Field(description="用户可以提出的后续问题")

```

## 返回结构化输出

定义了模式后，我们需要一种方法来指示模型使用它。
虽然一种方法是将此模式包含在提示中并*礼貌地请求*模型使用它，但不推荐这样做。
有几种更强大的方法可以利用模型提供商 API 中的原生功能。

### 使用工具调用

许多[模型提供商支持](/docs/integrations/chat/)工具调用，这一概念在我们[工具调用指南](/docs/concepts/tool_calling/)中有更详细的讨论。
简而言之，工具调用涉及将工具绑定到模型，并在适当的时候，模型可以*决定*调用此工具并确保其响应符合工具的模式。
考虑到这一点，核心概念很简单：*只需将我们的模式作为工具绑定到模型！*
以下是使用上面定义的 `ResponseFormatter` 模式的示例：

```python
from langchain_openai import ChatOpenAI
model = ChatOpenAI(model="gpt-4o", temperature=0)
# 将 responseformatter 模式作为工具绑定到模型
model_with_tools = model.bind_tools([ResponseFormatter])
# 调用模型
ai_msg = model_with_tools.invoke("什么是细胞的能量工厂？")
```

工具调用的参数已作为字典提取。
此字典可以根据需要解析为 Pydantic 对象，以匹配我们原始的 `ResponseFormatter` 模式。

```python
# 获取工具调用参数
ai_msg.tool_calls[0]["args"]
{'answer': "细胞的能量工厂是线粒体。线粒体是产生细胞大部分三磷酸腺苷 (ATP) 的细胞器，ATP 用作化学能的来源。",
 'followup_question': 'ATP 在细胞中的功能是什么？'}
# 将字典解析为 pydantic 对象
pydantic_object = ResponseFormatter.model_validate(ai_msg.tool_calls[0]["args"])
```

### JSON 模式

除了工具调用之外，一些模型提供商还支持称为“JSON 模式”的功能。
它支持将 JSON 模式定义作为输入，并强制模型生成符合要求的 JSON 输出。
您可以在[此处](/docs/integrations/chat/)找到支持 JSON 模式的模型提供商的表格。
以下是如何将 JSON 模式与 OpenAI 结合使用的示例：

```python
from langchain_openai import ChatOpenAI
model = ChatOpenAI(model="gpt-4o").with_structured_output(method="json_mode")
ai_msg = model.invoke("返回一个 JSON 对象，其中包含键 'random_ints' 和一个包含 10 个 [0-99] 范围内的随机整数的值")
ai_msg
{'random_ints': [45, 67, 12, 34, 89, 23, 78, 56, 90, 11]}
```

## 结构化输出方法

使用上述方法生成结构化输出时存在一些挑战：

1. 当使用工具调用时，需要将工具调用参数从字典解析回原始模式。
2. 此外，当我们需要强制执行结构化输出时，需要指示模型*始终*使用该工具，这是一个特定于提供商的设置。
3. 使用 JSON 模式时，需要将输出解析为 JSON 对象。

考虑到这些挑战，LangChain 提供了一个辅助函数 (`with_structured_output()`) 来简化该过程。

![结构化输出的图示](/img/with_structured_output.png)

它将模式作为工具绑定到模型，并将输出解析为指定的输出模式。

```python
# 将模式绑定到模型
model_with_structure = model.with_structured_output(ResponseFormatter)
# 调用模型
structured_output = model_with_structure.invoke("什么是细胞的能量工厂？")
# 获取 pydantic 对象
structured_output
ResponseFormatter(answer="细胞的能量工厂是线粒体。线粒体是产生细胞大部分三磷酸腺苷 (ATP) 的细胞器，ATP 用作化学能的来源。", followup_question='ATP 在细胞中的功能是什么？')
```
