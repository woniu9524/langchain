# LangChain 表达式语言 (LCEL)

:::info 先决条件
* [Runnable 接口](/docs/concepts/runnables)
:::

**L**ang**C**hain **E**xpression **L**anguage (LCEL) 采用一种 [声明式](https://en.wikipedia.org/wiki/Declarative_programming) 方法，通过现有的 Runnables 构建新的 [Runnables](/docs/concepts/runnables)。

这意味着您描述的是“应该”发生什么，而不是“如何”发生，从而使 LangChain 能够优化链的运行时执行。

我们通常将使用 LCEL 创建的 `Runnable` 称为“链”。请记住，“链”是一个 `Runnable`，它实现了完整的 [Runnable 接口](/docs/concepts/runnables)。

:::note
* [LCEL 速查表](/docs/how_to/lcel_cheatsheet/) 展示了涉及 Runnable 接口和 LCEL 表达式的常见模式。
* 请参阅以下 [操作指南](/docs/how_to/#langchain-expression-language-lcel) 列表，其中涵盖了 LCEL 的常见任务。
* 内置 `Runnables` 的列表可以在 [LangChain Core API 参考](https://python.langchain.com/api_reference/core/runnables.html) 中找到。在 LCEL 中使用这些 Runnables 组合自定义“链”时，其中许多 Runnables 都非常有用。
:::

## LCEL 的优势

LangChain 在许多方面优化了使用 LCEL 构建的链的运行时执行：

- **优化的并行执行**：使用 [RunnableParallel](#runnableparallel) 并行运行 Runnables，或使用 [Runnable Batch API](/docs/concepts/runnables/#optimized-parallel-execution-batch) 并行运行给定链的多个输入。并行执行可以显著减少延迟，因为处理可以并行而不是顺序进行。
- **保证的异步支持**：使用 [Runnable Async API](/docs/concepts/runnables/#asynchronous-support) 可以异步运行使用 LCEL 构建的任何链。这在使用需要并发处理大量请求的服务器环境时非常有用。
- **简化流式传输**：LCEL 链可以进行流式传输，允许在链执行时进行增量输出。LangChain 可以优化输出的流式传输，以最大限度地减少首次令牌时间（从 [聊天模型](/docs/concepts/chat_models) 或 [LLM](/docs/concepts/text_llms) 输出第一个块到第一个块出现之间经过的时间）。

其他优势包括：

- [**无缝的 LangSmith 跟踪**](https://docs.smith.langchain.com)
随着您的链变得越来越复杂，理解每一步到底发生了什么变得越来越重要。
使用 LCEL，**所有**步骤都会自动记录到 [LangSmith](https://docs.smith.langchain.com/)，以实现最大的可观察性和可调试性。
- **标准 API**：由于所有链都是使用 Runnable 接口构建的，因此它们可以与任何其他 Runnable 以相同的方式使用。
- [**可使用 LangServe 进行部署**](/docs/concepts/architecture#langserve)：使用 LCEL 构建的链可以使用 LangServe 进行生产部署。

## 我应该使用 LCEL 吗？

LCEL 是一个 [编排解决方案](https://en.wikipedia.org/wiki/Orchestration_(computing))——它允许 LangChain 以优化的方式处理链的运行时执行。

尽管我们看到用户在生产环境中运行了包含数百个步骤的链，但我们通常建议将 LCEL 用于更简单的编排任务。当应用程序需要复杂的有状态管理、分支、循环或多个代理时，我们建议用户利用 [LangGraph](/docs/concepts/architecture#langgraph)。

在 LangGraph 中，用户定义指定应用程序流程的图。这允许用户在 LCEL 需要时继续在单个节点中使用 LCEL，同时可以轻松定义更具可读性和可维护性的复杂编排逻辑。

以下是一些指导方针：

* 如果您正在进行单次 LLM 调用，则不需要 LCEL；而是直接调用底层的 [聊天模型](/docs/concepts/chat_models)。
* 如果您有一个简单的链（例如，prompt + llm + parser，简单的检索设置等），LCEL 是一个合理的选择，前提是您利用了 LCEL 的优势。
* 如果您正在构建一个复杂的链（例如，包含分支、循环、多个代理等），请改用 [LangGraph](/docs/concepts/architecture#langgraph)。请记住，您始终可以在 LangGraph 的单个节点中使用 LCEL。

## 组合原语

`LCEL` 链是通过组合现有的 `Runnables` 来构建的。两个主要的组合原语是 [RunnableSequence](https://python.langchain.com/api_reference/core/runnables/langchain_core.runnables.base.RunnableSequence.html#langchain_core.runnables.base.RunnableSequence) 和 [RunnableParallel](https://python.langchain.com/api_reference/core/runnables/langchain_core.runnables.base.RunnableParallel.html#langchain_core.runnables.base.RunnableParallel)。

许多其他组合原语（例如 [RunnableAssign](https://python.langchain.com/api_reference/core/runnables/langchain_core.runnables.passthrough.RunnableAssign.html#langchain_core.runnables.passthrough.RunnableAssign)）可以被视为这两个原语的变体。

:::note
您可以在 [LangChain Core API 参考](https://python.langchain.com/api_reference/core/runnables.html) 中找到所有组合原语的列表。
:::

### RunnableSequence

`RunnableSequence` 是一个组合原语，允许您将多个 runnables 顺序“链接”起来，一个 runnable 的输出作为下一个 runnable 的输入。

```python
from langchain_core.runnables import RunnableSequence
chain = RunnableSequence([runnable1, runnable2])
```

使用某些输入调用 `chain`：

```python
final_output = chain.invoke(some_input)
```

对应于以下内容：

```python
output1 = runnable1.invoke(some_input)
final_output = runnable2.invoke(output1)
```

:::note
`runnable1` 和 `runnable2` 是您想要链接在一起的任何 `Runnable` 的占位符。
:::

### RunnableParallel

`RunnableParallel` 是一个组合原语，允许您并发运行多个 runnables，并将相同的输入提供给每个 runnable。

```python
from langchain_core.runnables import RunnableParallel
chain = RunnableParallel({
    "key1": runnable1,
    "key2": runnable2,
})
```

使用某些输入调用 `chain`：

```python
final_output = chain.invoke(some_input)
```

将产生一个 `final_output` 字典，其键与输入字典相同，但值被相应 runnable 的输出替换。

```python
{
    "key1": runnable1.invoke(some_input),
    "key2": runnable2.invoke(some_input),
}
```

回想一下，runnables 是并行执行的，因此虽然结果与上面显示的字典推导相同，但执行时间要快得多。

:::note
`RunnableParallel` 支持同步和异步执行（正如所有 `Runnables` 所支持的那样）。

* 对于同步执行，`RunnableParallel` 使用 [ThreadPoolExecutor](https://docs.python.org/3/library/concurrent.futures.html#concurrent.futures.ThreadPoolExecutor) 来并发运行 runnables。
* 对于异步执行，`RunnableParallel` 使用 [asyncio.gather](https://docs.python.org/3/library/asyncio.html#asyncio.gather) 来并发运行 runnables。
:::

## 组合语法

`RunnableSequence` 和 `RunnableParallel` 的使用非常普遍，我们创建了使用它们的简写语法。这有助于使代码更具可读性和简洁性。

### `|` 运算符

我们已经 [重载](https://docs.python.org/3/reference/datamodel.html#special-method-names) 了 `|` 运算符，以从两个 `Runnables` 创建 `RunnableSequence`。

```python
chain = runnable1 | runnable2
```

等同于：

```python
chain = RunnableSequence([runnable1, runnable2])
```

### `.pipe` 方法

如果您对运算符重载有道德上的顾虑，可以使用 `.pipe` 方法代替。这等同于 `|` 运算符。

```python
chain = runnable1.pipe(runnable2)
```

### 类型强制转换

LCEL 应用自动类型强制转换，以便于组合链。

如果您不理解类型强制转换，可以随时直接使用 `RunnableSequence` 和 `RunnableParallel` 类。

这将使代码更冗长，但也会使其更明确。

#### 字典到 RunnableParallel

在 LCEL 表达式中，字典会自动转换为 `RunnableParallel`。

例如，以下代码：

```python
mapping = {
    "key1": runnable1,
    "key2": runnable2,
}

chain = mapping | runnable3
```

它会自动转换为以下内容：

```python
chain = RunnableSequence([RunnableParallel(mapping), runnable3])
```

:::caution
您必须小心，因为 `mapping` 字典不是 `RunnableParallel` 对象，它只是一个字典。这意味着以下代码将引发 `AttributeError`：

```python
mapping.invoke(some_input)
```
:::

#### 函数到 RunnableLambda

在 LCEL 表达式中，函数会自动转换为 `RunnableLambda`。

```
def some_func(x):
    return x

chain = some_func | runnable1
```

它会自动转换为以下内容：

```python
chain = RunnableSequence([RunnableLambda(some_func), runnable1])
```

:::caution
您必须小心，因为 lambda 函数不是 `RunnableLambda` 对象，它只是一个函数。这意味着以下代码将引发 `AttributeError`：

```python
lambda x: x + 1.invoke(some_input)
```
:::

## 旧版链

LCEL 旨在围绕行为和自定义提供一致性，以替代旧的子类化链，例如 `LLMChain` 和 `ConversationalRetrievalChain`。其中许多旧版链隐藏了重要的细节，例如提示，并且随着更多样化的可行模型出现，自定义变得越来越重要。

如果您当前正在使用其中一个旧版链，请参阅[此指南以获取有关如何迁移的指导](/docs/versions/migrating_chains)。

有关如何使用 LCEL 执行特定任务的指南，请查看[相关的操作指南](/docs/how_to/#langchain-expression-language-lcel)。
