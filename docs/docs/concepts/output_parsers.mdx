# 输出解析器

<span data-heading-keywords="output parser"></span>

:::note

此处的信息是指那些接收模型文本输出并尝试将其解析为更结构化表示的解析器。
越来越多的模型开始支持函数（或工具）调用，这可以自动处理此问题。
建议使用函数/工具调用而不是输出解析。
相关文档请参见[此处](/docs/concepts/tool_calling)。

:::

`Output parser` 负责接收模型的输出并将其转换为更适合下游任务的格式。
当您使用 LLM 生成结构化数据，或标准化聊天模型和 LLM 的输出时非常有用。

LangChain 提供了多种不同类型的输出解析器。以下是 LangChain 支持的输出解析器列表。下表包含各种信息：

- **名称 (Name)**：输出解析器的名称
- **支持流式输出 (Supports Streaming)**：该输出解析器是否支持流式输出。
- **包含格式说明 (Has Format Instructions)**：该输出解析器是否包含格式说明。这通常是可用的，除非 (a) 所需的模式不是在提示中指定的，而是在其他参数中指定的（例如 OpenAI 函数调用），或者 (b) 输出解析器包装了另一个输出解析器。
- **调用 LLM (Calls LLM)**：该输出解析器本身是否调用 LLM。这通常只由那些尝试纠正格式错误输出的输出解析器完成。
- **输入类型 (Input Type)**：预期的输入类型。大多数输出解析器都适用于字符串和消息，但有些（如 OpenAI Functions）需要带有特定关键字参数的消息。
- **输出类型 (Output Type)**：解析器返回对象的输出类型。
- **描述 (Description)**：我们对该输出解析器的评论以及何时使用它。
