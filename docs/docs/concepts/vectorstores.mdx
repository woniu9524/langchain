## 概述

向量存储是专门的数据存储，它能够根据向量表示对信息进行索引和检索。

这些被称为 [embeddings](/docs/concepts/embedding_models/) 的向量捕获了已嵌入数据的语义含义。

向量存储常用于搜索非结构化数据，如文本、图像和音频，以根据语义相似性而非精确的关键字匹配来检索相关信息。

## 集成

LangChain 拥有大量的向量存储集成，允许用户轻松地在不同的向量存储实现之间切换。

请参阅 [LangChain 向量存储集成的完整列表](/docs/integrations/vectorstores/)。

## 接口

LangChain 提供了一个标准接口来处理向量存储，允许用户轻松地在不同的向量存储实现之间切换。

该接口包含用于在向量存储中写入、删除和搜索文档的基本方法。

关键方法包括：

- `add_documents`: 将文本列表添加到向量存储中。
- `delete`: 从向量存储中删除文档列表。
- `similarity_search`: 搜索与给定查询相似的文档。


## 初始化

LangChain 中的大多数向量存储在初始化向量存储时接受一个嵌入模型作为参数。

我们将使用 LangChain 的 [InMemoryVectorStore](https://python.langchain.com/api_reference/core/vectorstores/langchain_core.vectorstores.in_memory.InMemoryVectorStore.html) 实现来演示 API。

```python
from langchain_core.vectorstores import InMemoryVectorStore
# 使用嵌入模型初始化
vector_store = InMemoryVectorStore(embedding=SomeEmbeddingModel())
```

## 添加文档

要添加文档，请使用 `add_documents` 方法。

此 API 使用 [Document](https://python.langchain.com/api_reference/core/documents/langchain_core.documents.base.Document.html) 对象列表。
`Document` 对象都具有 `page_content` 和 `metadata` 属性，这是一种存储非结构化文本和相关元数据的通用方式。

```python
from langchain_core.documents import Document

document_1 = Document(
    page_content="I had chocolate chip pancakes and scrambled eggs for breakfast this morning.",
    metadata={"source": "tweet"},
)

document_2 = Document(
    page_content="The weather forecast for tomorrow is cloudy and overcast, with a high of 62 degrees.",
    metadata={"source": "news"},
)

documents = [document_1, document_2]

vector_store.add_documents(documents=documents)
```

您通常应该为添加到向量存储中的文档提供 ID，这样
您就可以更新现有文档，而不是多次添加同一文档。

```python
vector_store.add_documents(documents=documents, ids=["doc1", "doc2"])
```

## 删除

要删除文档，请使用 `delete` 方法，该方法接受要删除的文档 ID 列表。

```python
vector_store.delete(ids=["doc1"])
```

## 搜索

向量存储会嵌入并存储已添加的文档。
如果我们传入一个查询，向量存储将嵌入该查询，在嵌入的文档上执行相似性搜索，并返回最相似的文档。
这包含了两个重要概念：首先，需要有一种方法来衡量查询与*任何* [嵌入](/docs/concepts/embedding_models/) 文档之间的相似性。
其次，需要有一种算法来有效地对*所有*嵌入的文档执行此相似性搜索。

### 相似性度量

嵌入向量的一个关键优势在于它们可以使用许多简单的数学运算进行比较：

- **余弦相似度**: 测量两个向量之间夹角的余弦值。
- **欧氏距离**: 测量两点之间的直线距离。
- **点积**: 测量一个向量在另一个向量上的投影。

相似性度量的选择有时可以在初始化向量存储时进行选择。请参阅
您正在使用的特定向量存储的文档，以了解支持哪些相似性度量。

:::info[Further reading]

* 请参阅 Google 关于嵌入相似性度量的[这篇文档](https://developers.google.com/machine-learning/clustering/dnn-clustering/supervised-similarity)。
* 请参阅 Pinecone 关于相似性度量的[博客文章](https://www.pinecone.io/learn/vector-similarity/)。
* 请参阅 OpenAI 关于使用 OpenAI 嵌入时应使用哪种相似性度量的[常见问题解答](https://platform.openai.com/docs/guides/embeddings/faq)。

:::

### 相似性搜索

给定一个用于衡量嵌入查询与任何嵌入文档之间距离的相似性度量，我们需要一种算法来有效地搜索*所有*嵌入的文档，以找到最相似的文档。
有多种方法可以做到这一点。例如，许多向量存储实现了 [HNSW（分层可导航小世界）](https://www.pinecone.io/learn/series/faiss/hnsw/)，这是一种基于图的索引结构，可以进行高效的相似性搜索。
无论底层使用何种搜索算法，LangChain 向量存储接口都有一个适用于所有集成的 `similarity_search` 方法。
这将接收搜索查询，创建嵌入，查找相似文档，并将它们作为 [Documents](https://python.langchain.com/api_reference/core/documents/langchain_core.documents.base.Document.html) 列表返回。

```python
query = "my query"
docs = vectorstore.similarity_search(query)
```

许多向量存储支持通过 `similarity_search` 方法传递搜索参数。请参阅您正在使用的特定向量存储的文档，以了解支持哪些参数。
例如，[Pinecone](https://python.langchain.com/api_reference/pinecone/vectorstores/langchain_pinecone.vectorstores.PineconeVectorStore.html#langchain_pinecone.vectorstores.PineconeVectorStore.similarity_search) 提供了几个重要的通用概念参数：
许多向量存储支持 [ `k` ](/docs/integrations/vectorstores/pinecone/#query-directly) 参数，该参数控制返回的文档数量，以及 `filter` 参数，该参数允许按元数据过滤文档。

- `query (str) - 要查找相似文档的文本。`
- `k (int) - 要返回的文档数量。默认为 4。`
- `filter (dict | None) - 用于过滤元数据的参数字典`

:::info[Further reading]

* 有关如何使用 `similarity_search` 方法的更多详细信息，请参阅[操作指南](/docs/how_to/vectorstores/)。
* 有关可以传递给特定向量存储的 `similarity_search` 方法的参数的更多详细信息，请参阅[集成页面](/docs/integrations/vectorstores/)。

:::

### 元数据过滤

虽然向量存储实现了搜索算法来高效地搜索*所有*嵌入的文档以找到最相似的文档，但许多向量存储还支持按元数据进行过滤。
元数据过滤通过应用特定条件（例如，检索来自特定来源或日期范围的文档）来帮助缩小搜索范围。这两个概念协同工作得很好：

1. **语义搜索**: 直接查询非结构化数据，通常通过嵌入或关键字相似性。
2. **元数据搜索**: 将结构化查询应用于元数据，过滤特定文档。

向量存储对元数据过滤的支持通常取决于底层的向量存储实现。

以下是使用 [Pinecone](/docs/integrations/vectorstores/pinecone/#query-directly) 的示例用法，其中我们过滤所有具有值为 `tweet` 的元数据键 `source` 的文档。

```python
vectorstore.similarity_search(
    "LangChain provides abstractions to make working with LLMs easy",
    k=2,
    filter={"source": "tweet"},
)
```

:::info[Further reading]

* 请参阅 Pinecone 关于使用元数据进行过滤的[文档](https://docs.pinecone.io/guides/data/filter-with-metadata)。
* 请参阅支持元数据过滤的[LangChain 向量存储集成列表](/docs/integrations/retrievers/self_query/)。

:::

## 高级搜索和检索技术

虽然 HNSW 等算法在许多情况下为高效的相似性搜索奠定了基础，但可以采用其他技术来提高搜索质量和多样性。
例如，[最大边际相关性](https://python.langchain.com/v0.1/docs/modules/model_io/prompts/example_selectors/mmr/) 是一种重新排序算法，用于使搜索结果多样化，它在初始相似性搜索之后应用，以确保结果集更加多样化。
第二个例子是，一些 [向量存储](/docs/integrations/retrievers/pinecone_hybrid_search/) 提供内置的 [混合搜索](https://docs.pinecone.io/guides/data/understanding-hybrid-search)，以结合关键字和语义相似性搜索，从而结合了这两种方法的优点。
目前，没有统一的方法可以使用 LangChain 向量存储执行混合搜索，但它通常作为通过 `similarity_search` 传递的关键字参数暴露出来。
有关混合搜索的更多详细信息，请参阅此[操作指南](/docs/how_to/hybrid/)。

|-------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------|
| [混合搜索](/docs/integrations/retrievers/pinecone_hybrid_search/)                                            | 在结合基于关键字的搜索和语义相似性时。 | 混合搜索结合了关键字搜索和语义相似性，结合了这两种方法的优点。[论文](https://arxiv.org/abs/2210.11934)。 |
