# Agent (代理)

语言模型本身无法执行操作，它们只能输出文本。Agent 是一种系统，它接收一个高级任务，并使用 LLM 作为推理引擎来决定要执行什么操作以及如何执行这些操作。

[LangGraph](/docs/concepts/architecture#langgraph) 是 LangChain 的一个扩展，专门用于创建高度可控和可定制的 Agent。我们建议您使用 LangGraph 来构建 Agent。

有关更多信息，请参阅以下资源：

* LangGraph 关于[常见 Agent 架构](https://langchain-ai.github.io/langgraph/concepts/agentic_concepts/)的文档
* [LangGraph 中的预构建 Agent](https://langchain-ai.github.io/langgraph/reference/prebuilt/#langgraph.prebuilt.chat_agent_executor.create_react_agent)

## 旧版 Agent 概念：AgentExecutor

LangChain 之前推出了 `AgentExecutor` 作为 Agent 的运行时。
虽然它是一个极佳的起点，但在处理更复杂和定制化的 Agent 时，其局限性逐渐显现。
因此，我们正在逐步淘汰 `AgentExecutor`，转而采用 LangGraph 中更灵活的解决方案。

### 从 AgentExecutor 过渡到 LangGraph

如果您目前正在使用 `AgentExecutor`，请不用担心！我们已准备好相关资源来帮助您：

1. 对于仍需要使用 `AgentExecutor` 的用户，我们提供了一份关于[如何使用 AgentExecutor](/docs/how_to/agent_executor) 的综合指南。

2. 然而，我们强烈建议您过渡到 LangGraph 以获得更好的灵活性和控制力。为了促进这一过渡，我们创建了一个详细的[迁移指南](/docs/how_to/migrate_agent)，帮助您无缝地从 `AgentExecutor` 迁移到 LangGraph。
