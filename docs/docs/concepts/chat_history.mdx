# 对话历史

:::info 先决条件

- [消息](/docs/concepts/messages)
- [聊天模型](/docs/concepts/chat_models)
- [工具调用](/docs/concepts/tool_calling)
:::

对话历史是用户与聊天模型之间对话的记录。它用于在整个对话中维护上下文和状态。对话历史是一系列[消息](/docs/concepts/messages)，每条消息都与一个特定的[角色](/docs/concepts/messages#role)相关联，例如“user”（用户）、“assistant”（助手）、“system”（系统）或“tool”（工具）。

## 对话模式

![对话模式](/img/conversation_patterns.png)

大多数对话都以一个**系统消息**开始，该消息设置对话的上下文。接着是包含用户输入的**用户消息**，然后是包含模型响应的**助手消息**。

**助手**可以直接响应用户，或者如果配置了工具，则请求调用[工具](/docs/concepts/tool_calling)来执行特定任务。

完整的对话通常涉及两种交替消息模式的组合：

1.  **用户**和**助手**代表来回的对话。
2.  **助手**和**工具消息**代表一个["代理"工作流](/docs/concepts/agents)，其中助手调用工具来执行特定任务。

## 管理对话历史

由于聊天模型对输入大小有最大限制，因此管理对话历史并根据需要进行修剪以避免超出[上下文窗口](/docs/concepts/chat_models/#context-window)非常重要。

在处理对话历史时，务必保持正确的对话结构。

管理对话历史的关键指南：

-   对话应遵循以下结构之一：
    -   第一条消息是“user”消息或“system”消息，后跟一条“user”消息，然后是一条“assistant”消息。
    -   最后一条消息应是“user”消息或包含工具调用结果的“tool”消息。
-   在使用[工具调用](/docs/concepts/tool_calling)时，“tool”消息只能跟在请求工具调用的“assistant”消息之后。

:::tip
理解正确的对话结构对于能够正确实现聊天模型的[记忆](https://langchain-ai.github.io/langgraph/concepts/memory/)至关重要。
:::
