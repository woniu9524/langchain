# Langchain 中的异步编程

:::info 先决条件
* [Runnable 接口](/docs/concepts/runnables)
* [asyncio](https://docs.python.org/3/library/asyncio.html)
:::

基于 LLM 的应用程序通常涉及大量 I/O 密集型操作，例如调用语言模型、数据库或其他服务的 API。异步编程（或 async programming）是一种允许程序并发执行多个任务而不阻塞其他任务执行的范例，从而提高效率和响应能力，尤其是在 I/O 密集型操作中。

:::note
您应该熟悉 Python 中的异步编程，然后再阅读本指南。如果您不熟悉，请在线查找合适的资源来学习如何在 Python 中进行异步编程。
本指南专门关注您在异步环境中处理 LangChain 时需要了解的内容，并假设您已经熟悉异步编程。
:::

## Langchain 异步 API

许多 LangChain API 都设计为异步的，允许您构建高效且响应迅速的应用程序。

通常，任何可能执行 I/O 操作的方法（例如，调用 API、读取文件）都会有一个异步的对应方法。

在 LangChain 中，异步实现位于与其同步对应方法相同的类中，异步方法有一个“a”前缀。例如，同步的 `invoke` 方法有一个名为 `ainvoke` 的异步对应方法。

LangChain 的许多组件都实现了 [Runnable 接口](/docs/concepts/runnables)，该接口支持异步执行。这意味着您可以使用 Python 中的 `await` 关键字异步运行 Runnables。

```python
await some_runnable.ainvoke(some_input)
```

其他不实现 [Runnable 接口](/docs/concepts/runnables) 的组件，如 [Embedding Models](/docs/concepts/embedding_models) 和 [VectorStore](/docs/concepts/vectorstores)，通常也遵循相同的规则，并在同一类中包含方法的异步版本，并带有“a”前缀。

例如：

```python
await some_vectorstore.aadd_documents(documents)
```

使用 [LangChain Expression Language (LCEL)](/docs/concepts/lcel) 创建的 Runnables 也可以异步运行，因为它们实现了完整的 [Runnable 接口](/docs/concepts/runnables)。

有关更多信息，请参阅您正在使用的特定组件的 [API 参考](https://python.langchain.com/api_reference/)。

## 委托给同步方法

大多数流行的 LangChain 集成都实现了其 API 的异步支持。例如，许多 ChatModel 实现的 `ainvoke` 方法使用 `httpx.AsyncClient` 来向模型提供商的 API 发出异步 HTTP 请求。

当没有可用的异步实现时，LangChain 会尝试提供一个默认实现，即使这会产生**轻微**的开销。

默认情况下，LangChain 会将未实现异步方法的执行委托给其同步对应方法。LangChain 几乎总是假设同步方法应被视为阻塞操作，并应在单独的线程中运行。
这是使用 `asyncio` 库提供的 [asyncio.loop.run_in_executor](https://docs.python.org/3/library/asyncio-eventloop.html#asyncio.loop.run_in_executor) 功能完成的。LangChain 使用 `asyncio` 库提供的默认执行器，该执行器使用线程池执行器惰性初始化，并具有在给定事件循环中重用的默认线程数。虽然此策略会因线程之间的上下文切换而产生轻微开销，但它保证了每个异步方法都有一个开箱即用的默认实现。

## 性能

LangChain 中的异步代码通常应该表现良好，开销最小，并且在大多数应用程序中不太可能成为瓶颈。

开销的两个主要来源是：

1.  [委托给同步方法](#delegation-to-sync-methods)时线程之间上下文切换的成本。这可以通过提供原生的异步实现来解决。
2.  在 [LCEL](/docs/concepts/lcel) 中，作为链一部分出现的任何“廉价函数”将被安排为事件循环上的任务（如果它们是异步的）或在单独的线程中运行（如果它们是同步的），而不是仅仅内联运行。

您应该预期的延迟开销在几十微秒到几毫秒之间。

更常见的性能问题源于用户意外地通过在异步上下文中调用同步代码（例如，调用 `invoke` 而不是 `ainvoke`）来阻塞事件循环。

## 兼容性

LangChain 仅与 `asyncio` 库兼容，该库作为 Python 标准库的一部分分发。它不适用于 `trio` 或 `curio` 等其他异步库。

在 Python 3.9 和 3.10 中，[asyncio 的任务](https://docs.python.org/3/library/asyncio-task.html#asyncio.create_task)不接受 `context` 参数。由于此限制，LangChain 在某些情况下无法自动将 `RunnableConfig` 传播到调用链的下游。

如果您在使用 Python 3.9 或 3.10 时遇到异步代码中的流式传输、回调或跟踪问题，这很可能是原因。

请阅读 [RunnableConfig 的传播](/docs/concepts/runnables/#propagation-of-runnableconfig) 以了解更多详细信息，了解如何手动将 `RunnableConfig` 传播到调用链的下游（或升级到不再存在此问题的 Python 3.11）。

## 如何在 ipython 和 jupyter notebooks 中使用

从 IPython 7.0 开始，IPython 支持异步 REPL。这意味着您可以在 IPython REPL 和 Jupyter Notebook 中使用 `await` 关键字，而无需任何额外的设置。有关更多信息，请参阅 [IPython 博客文章](https://blog.jupyter.org/ipython-7-0-async-repl-a35ce050f7f7)。
